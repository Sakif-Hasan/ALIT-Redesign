# ALIT-Redesign

An entirely new, ground-up website redesign built with a focus on accessibility, compliance, and a scalable architectural foundation. 

> **Note:** This repository is strictly for development purposes.

---

## Project Scope
The goal of this project is to rebuild our web presence from scratch. Every component and feature must be engineered to meet modern web standards, strict accessibility (WCAG) guidelines, and regulatory compliance while maintaining a robust, future-proof codebase.

---

## Contribution Rules

To maintain codebase integrity and ensure smooth collaboration, all contributors must adhere to the following workflow:

* **Feature Branching:** Always create a dedicated branch for your work. Do not commit directly to the `main` branch.
* **Peer Review:** Every merge request to the `main` branch requires at least one peer review and approval.
* **Testing:** Code must be thoroughly tested locally before submission to ensure no major faults or regressions are introduced.
* **Commit Naming Convention:** Every commit must strictly follow this format to allow for clean tracking and easy rollbacks:
    ```text
    [FirstLetterOfName]_[CommitNumber]_[CommitName]
    ```
    *Example:* `S_04_AddedAccessibleNavbar`
